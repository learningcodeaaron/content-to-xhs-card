# 布局目录与选择规则

先判断内容关系，再选择布局。不要按“这组卡还缺一种样式”选择。

| 布局 ID | 适合内容 | 必要字段 | 不应使用 |
| --- | --- | --- | --- |
| `cover.editorial` | 结论型封面 | `title/subtitle/quote` | 长正文 |
| `cover.image` | 人物、产品、对象明确的封面 | `title/quote` + 图片 | 没有合适图片时强塞空图 |
| `statement.poster` | 单一判断、定义、概念 | `title/lead/quote` | 多组并列关系 |
| `comparison.split` | 同维度 A/B 对照 | `left/right` | 三个以上对象 |
| `process.timeline` | 3–6 个先后阶段 | `items[]` | 没有顺序的列表 |
| `process.cycle` | 持续反馈、复盘闭环 | `items[]` | 单向流程 |
| `framework.axes` | 2–5 组二元维度 | `axes[]` | 带精确数值的排名 |
| `framework.matrix` | 象限、分类、4×4 组合 | `matrix[]` | 用格子面积暗示不存在的规模 |
| `framework.bento` | 一个核心加若干支撑模块 | `items[]` | 各项完全同权却故意做大小块 |
| `data.ranking` | 同单位、顺序有意义的数值 | `stats[].value` | 只有观点没有数据 |
| `data.dashboard` | 指标、构成、对比数据 | `stats[]` | 伪造百分比、趋势或强弱 |
| `list.cards` | 3–6 个完整要点 | `items[]` | 可以明确表达流程/对比/矩阵时 |
| `quote.portrait` | 有明确人物归属的观点 | `quote` + 人物图 | 没有人物来源的普通金句 |
| `tail.cta` | 收束和关注入口 | `quote/cta` | 新增一段长论证 |

## 各布局怎么填内容(关键)

- **`statement.poster`(单命题海报)**:纯文字版式,**没有中心图形**。结构 = 大标题(判断)+ 一句 `lead`(展开解释)+ 一个 `quote` 强调金句面板。适合"有用≠有护城河""技术领先未必稳"这类单一判断。不要指望它显示 `items`;有并列要点应改用 `list.cards`。
- **`cover.editorial`(封面)**:大标题 + `subtitle` + `quote` 核心判断面板。无人物图时不放任何占位图形,金句面板占据下半屏。
- **`comparison.split`(对照)**:必须提供 `left` 和 `right` 两个对象,各含 `label`(胶囊标签,如"容易被吃掉"/"更有价值")、`title`、`items[]`(每项一句完整判断)。底部 `quote` 给对照结论。仅用于**同一维度**的两方对比。
- **`list.cards`(要点卡组)**:`items[]` 放 3–4 个并列、无顺序的完整要点,每项 `title`(判断)+ `body`(解释)。用于"四类护城河""五个要问的问题"这类清单。
- **`process.timeline`(阶段链)**:`items[]` 表达**有先后/因果顺序**的 3–6 步。用于展示一条推理链或递进逻辑(如"AI替代白领→Agent是核心→企业付费→长成巨头")。

## 组合约束

- 一张卡只保留一个主关系；标题、主视觉、收束句分别回答“结论、关系、记忆点”。
- 6 张以上卡组通常至少覆盖 3 个布局族。
- 流程最多 6 节点；对比每侧最多 5 项；Bento 最多 5 个主要块；矩阵最多 16 格；排名最多 8 行。
- 二元维度没有数值时使用轴或矩阵，不使用柱状图。
- 排名必须有可比较的同单位数值；环图必须是能合计为整体的构成数据。
- 图中文字保留为可编辑文字；SVG 只表达线、轴、箭头、节点与几何关系。
- **严禁装饰性图形**:任何图形/SVG 都必须承载本卡独有的真实信息。若图内文字换成任意词都不影响理解,或多张卡出现同一组图形词(如"输入/理解/行动"),即为噪音,一律删除。
