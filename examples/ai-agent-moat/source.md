## “技术领先不是护城河。技术只有在转化为更低的成本、更强的品牌、更高的转换成本或者网络效应以后，才有可能成为护城河。”

这句话放在 AI Agent 公司身上，我觉得特别合适。

现在 AI Agent 很火，火到什么程度呢？以前大家讨论 AI 产品，还是 ChatGPT、Claude、DeepSeek、豆包、Kimi 这些大模型入口；现在讨论 AI 创业，很多人张口闭口就是 Agent，办公 Agent、销售 Agent、客服 Agent、投研 Agent、代码 Agent。

听起来都很性感。

因为 Agent 这个词天然就带着一种想象力：它能自己拆任务、调工具、交付结果。往大了说，它像是一个数字员工；往小了说，它至少像一个更能干的自动化助手。

如果这个东西真的跑通，那当然是很大的机会。

但问题来了。

## AI Agent 公司到底有没有护城河？

这个问题不能靠一句“Agent 是未来”来回答。AI 会不会改变工作方式，这个答案大概率是会的；Agent 会不会成为未来软件的重要形态，这个答案我也倾向于是会的。

真正困难的地方不是判断 Agent 有没有未来。

真正困难的是：今天这些做 Agent 的公司，3 年、5 年、10 年后，到底靠什么长期赚钱？

## 先问一个最朴素的问题

问你一个问题。

如果你开了一家公司，给客户做一个 AI Agent，帮他处理销售线索、整理客户信息、生成跟进话术、自动写邮件、推送商机提醒。

客户用了以后发现，确实有用。

那这是不是一个好生意？

不一定。

因为“有用”和“有护城河”是两件事。

一个产品有用，只能说明它解决了某个问题。一个公司有护城河，说明别人没那么容易解决同样的问题，或者即便也能解决，客户也不容易换掉你。

这中间差得非常远。

就像一个餐馆好吃，不代表它就是好生意。你二舅在小区门口开个餐馆，味道不错，附近居民也爱吃，但隔壁再开一家差不多的，厨师挖一挖、菜单抄一抄、价格便宜一点，客流可能就被分走了。

你说它有没有价值？

当然有。

但你要问它是不是能支撑 10 年高利润的好生意，那就得看房租、位置、厨师、供应链、复购、品牌、管理半径。

AI Agent 公司也是一样。

能做出一个有用的 Agent，不等于做出了一门有护城河的生意。

## Agent 公司最容易被高估的地方

现在很多 Agent 公司的投资逻辑，大概是这样的：

1. AI 会替代大量白领流程；
2. Agent 是 AI 落地的核心形态；
3. 企业愿意为效率提升付费；
4. 所以做 Agent 的公司会成为下一代软件巨头。

这个逻辑听起来很顺。

但这里面最大的假设是什么？

Agent 公司能把“模型能力”包装成“客户离不开的产品能力”。

这个假设很关键。

因为如果 Agent 公司只是把大模型包了一层壳，接几个 API，做一些工作流编排，再加一个好看的界面，那

这个东西很难有长期护城河。

今天你能接 Claude，明天别人也能接。今天你能接 GPT，明天别人也能接。今天你能做一个销售 Agent，明天 CRM 厂商自己也能做。今天你能做一个投研 Agent，明天券商、基金、数据服务商、金融终端都能做。

你真正要问的是：

客户为什么非得用你？

不是为什么现在觉得你不错。

而是为什么 3 年后、5 年后，当模型越来越便宜、工具调用越来越标准、工作流编排越来越容易的时候，他还必须继续用你。

这才是护城河问题。

## 技术领先，在 Agent 这里可能更不稳定

很多人会说，我的 Agent 效果更好。

这当然重要。

但效果更好是不是护城河，要看它靠什么实现。

如果靠的是你调 Prompt 调得更细，靠的是你把几个工具串得更顺，靠的是你比别人早半年理解某个场景，那这个领先未必稳。

因为 Agent 这个领域有一个很麻烦的地方：

它的底层能力进步太快了。

大模型自己在变强。

工具调用在变强。

多模态在变强。

长上下文在变强。

浏览器操作、代码执行、文件理解、表格处理、企业知识库调用，这些能力都在变成基础设施。

这会带来一个结果：很多今天看起来需要一个创业公司认真解决的问题，明天可能被模型厂商顺手做掉。

你今天做了一个“自动读邮件、总结重点、生成回复”的 Agent，明天 Google、Microsoft、腾讯、飞书、钉钉、WPS 都可以把它做进自己的办公套件里。

你今天做了一个“自动看 CRM、生成销售跟进计划”的 Agent，明天 Salesforce、HubSpot、纷享销客、销售易也可以做。

你今天做了一个“自动读代码库、修 bug、写测试”的 Agent，明天 Claude Code、Codex、Cursor、JetBrains、GitHub Copilot 都会继续往前卷。

所以，Agent 公司如果只靠“我比别人更会用模型”，这个护城河是不够的。

技术领先当然有价值，但它不是最终护城河。

## 真正可能形成护城河的，只有几类

我觉得 AI Agent 公司真正可能有护城河，大概只有几种情况。

## 第一种，掌握客户的深层业务流程

这类公司不是简单做一个工具，而是嵌进客户的关键流程里。

比如企业的销售线索流转、合同审批、客服工单、投研流程、供应链计划、风控审核、保险理赔、医疗病历处理。

如果一个 Agent 只是帮你写两段话，那客户换掉它很容易。

但如果一个 Agent 已经接入了你的系统、权限、数据、审批流、历史记录、团队协作方式，还沉淀了大量任务执行记录，那它就不再只是一个聊天框。

它变成了流程的一部分。

这时候客户切换成本会提高。

但这里面也有问题。

能嵌进流程的公司，往往是原来就掌握流程入口的公司。

比如飞书、钉钉、企业微信、Microsoft 365、Salesforce、SAP、ServiceNow、WPS、各类垂直 SaaS。

他们本来就在客户工作流里。

Agent 对他们来说，是给已有系统加智能层；对创业公司来说，是要先打进客户流程，再建立信任，再接系统，再拿数据，再证明价值。

这个难度不是一个级别的。

所以，流程型护城河不是没有，但它更容易被已有软件巨头拿到。

创业公司要想拿到，就必须在某个非常具体、非常复杂、巨头暂时顾不上的垂直场景里，做得足够深。

## 第二种，掌握独家数据

Agent 要做得好，光有通用模型不够。

它得知道你的客户是谁、过去怎么沟通、合同怎么签、哪些线索容易成交、哪些问题经常出错、哪个审批节点卡得最久、哪个销售最会转化、哪个客服最容易把客户聊炸。

这些东西不是通用知识。

这是企业自己的动态数据。

如果一家 Agent 公司能长期服务某类客户，沉淀大量行业数据、任务数据、反馈数据，并且这些数据能持续改善产品效果，那它可能会形成护城河。

但这里也要小心。

企业数据分两种。

一种是行业通识。比如保险理赔流程怎么走，财务报销有哪些规则，销售跟进一般怎么写，客服问题怎么分类。这些东西一开始看起来有行业 know-how，但时间一长，很容易被大模型、咨询公司、行业软件、开源模板慢慢吃掉。

另一种是客户自己的个性化、时效性数据。比如某家公司真实客户行为、内部流程、历史成交记录、实时库存、组织权限、员工习惯、用户画像。

后者才更有价值。

所以 Agent 公司如果说自己有“行业数据护城河”，你得继续问一句：

这些数据到底是行业通识，还是客户独家的动态数据？

如果只是行业通识，护城河大概率没那么深。

## 第三种，形成很高的切换成本

软件公司最舒服的护城河之一，就是客户懒得换。

不是客户爱你爱到不行，而是换你太麻烦。

比如数据迁移麻烦，员工培训麻烦，流程重配麻烦，历史记录丢失麻烦，系统接口重接麻烦，内部审批重新走一遍更麻烦。

Agent 公司如果能形成这种切换成本，它就有机会成为好生意。

但问题是，很多 Agent 产品现在还停留在“外挂”阶段。

今天用户打开它，让它帮忙写一段话、搜一份资料、跑一次分析。用完就走。

这类产品切换成本很低。

A 家不好用，我换 B 家。B 家贵了，我换 C 家。模型公司出了原生功能，我直接用原生。

真正有切换成本的 Agent，必须管理任务状态、连接业务系统、保存执行记录、理解组织权限、承接多人协作，还要让客户的历史流程逐渐围绕它运转。

这个时候，它才开始像一门软件生意。

否则，它更像一个模型能力展示台。

## 第四种，拿到分发入口

2C Agent 更残酷。

如果一个 Agent 面向个人用户，帮你订票、购物、写邮件、做计划、管理日程、搜索网页，那它的竞争对手不是其他创业公司，而是操作系统、浏览器、手机厂商、超级 App 和模型入口。

苹果、Google、Microsoft、OpenAI、Anthropic、Meta、腾讯、字节，这些公司天然离用户更近。

你做一个个人 Agent，用户凭什么每天打开你？

这个问题非常现实。

2C 产品的护城河，很多时候不是功能，而是入口、习惯和网络效应。

微信不是因为每个功能都最好，所以大家不走。它是因为你的关系链、支付、群聊、小程序、视频号、公众号、文件、工作沟通全在里面。

Google 搜索也不是简单一个搜索框，它背后是默认入口、广告系统、用户习惯、浏览器、安卓、海量索引和商业闭环。

个人 Agent 如果拿不到入口，很容易变成一个“好用但不常用”的产品。

这个状态最尴尬。

用户夸你很好，但不打开；打开过几次，但不续费；愿意试用，但不愿意长期依赖。

这就不是好生意。

## 模型公司会不会亲自下场？

这可能是 Agent 公司最大的风险之一。

如果 Agent 公司做的场景足够大、利润足够厚，模型公司为什么不自己做？

OpenAI 不会只满足于提供 API，Anthropic 也不会只满足于卖 Claude。它们一定会继续往代码、办公、企业知识库、浏览器、工作流、搜索、Agent 平台里走。

这个逻辑很简单。

大模型在 AI 应用链条里创造的价值太厚了。模型公司会想，既然用户最后是为了完成任务付费，那我为什么只赚底层推理的钱？我也可以包一层产品，把上层利润一起拿走。

这对独立 Agent 公司是很大的压力。

你如果做得太小，市场空间不够，长不大。

你如果做得太大，模型公司、平台公司、SaaS 巨头都会盯上你。

这就是很难受的地方。

所以，独立 Agent 公司真正舒服的位置，可能不是“做一个万能 Agent”，而是在一个足够深、足够脏、足够具体、巨头短期不愿意亲自下场的业务流程里，帮客户把事情跑通。

比如某类制造业供应链、某类金融运营流程、某类医疗文书、某类法律审查、某类跨境电商运营、某类政企流程。

但这种公司就不会那么性感。

它更像垂直软件公司，不像下一个 OpenAI。

## Agent 公司的护城河，不能只看 Demo

现在很多 Agent 产品的 Demo 都很好看。

输入一个目标，它自己拆任务；打开网页，查资料；调表格，写总结；生成 PPT；自动发邮件；甚至还能自己修 bug。

看起来很像未来。

但投资不能只看 Demo。

Demo 解决的是“能不能做出来”。

生意解决的是“能不能长期赚钱”。

这里面至少还要问几个问题：

客户是不是愿意持续付费？

付费金额能不能覆盖模型成本、交付成本、销售成本和售后成本？

客户使用越久，产品是不是越好用？

客户换掉你会不会很麻烦？

模型变强以后，你的价值是被增强，还是被吞掉？

大平台下场以后，你还剩什么？

这几个问题回答不出来，只说 Agent 是未来，是没有意义的。

就像电动车当然是未来，但不是每一家电动车公司都是好生意。光伏当然是未来，但不妨碍很多光伏企业卷到利润很薄。互联网当然是未来，但当年活下来的也就那么几家。

行业方向对，不代表每家公司都有护城河。

## 我比较愿意相信哪类 Agent 公司？

如果一定要选，我会更愿意相信几类。

## 第一类，已经掌握企业入口的平台型公司。

比如办公套件、企业 IM、CRM、ERP、云厂商、代码平台。它们本来就在流程里，Agent 是增强，不是从零获客。

## 第二类，深度垂直行业流程公司。

它不追求做万能 Agent，而是把某个行业里特别麻烦、特别高频、特别有付费能力的流程吃透。客户不是为了体验新技术买单，而是为了少出错、少招人、少返工、多成交。

## 第三类，拥有独家动态数据的公司。

比如本来就拥有用户画像、交易数据、内容分发数据、驾驶数据、供应链数据、企业协作数据的公司。Agent 对它们来说，是释放数据价值的方式。

## 第四类，能把 Agent 变成系统，而不是工具的公司。

工具用完就走，系统会留下任务状态、权限、历史记录、协作关系和流程依赖。

这两者差别很大。

## 最危险的是哪类？

最危险的是这类公司：

- 没有自己的模型；
- 没有独家数据；
- 没有客户入口；
- 没有深度流程；
- 没有高切换成本；
- 主要靠 Prompt、工作流编排和界面体验；
- 客户还觉得它是一个可选插件，而不是业务系统。

这种 Agent 公司短期可能增长很快，Demo 也会很好看，融资故事也很好讲。

但它的长期利润很难看清。

因为它的价值很可能被三边挤压：

上游模型公司能力越来越强，往下做应用；

旁边平台公司掌握客户入口，顺手加 Agent；

同行创业公司接同样的模型、做类似场景，开始价格竞争。

你说它能不能赚钱？

当然能。

尤其在早期市场教育阶段，很多客户愿意试，很多企业也有预算。

但能赚钱和有护城河不是一回事。

就像很多外包公司也赚钱，很多咨询公司也赚钱，很多系统集成商也赚钱，但这类生意往往人效、交付、销售、项目制、客户定制都会限制长期利润率。

如果 Agent 公司最后变成“AI 时代的软件外包 + 咨询交付”，那就不是一个特别性感的生意。

## 说回主题

## AI Agent 公司到底有没有护城河？

我的答案是：

有，但不在 Agent 这三个字里。

Agent 只是产品形态，不是护城河。

真正的护城河可能在流程里、数据里、入口里、切换成本里、行业 know-how 里、客户关系里、系统集成深度里。

如果一家公司只是因为会做 Agent 就说自己有护城河，那这个逻辑大概率站不住。

如果一家公司原本就掌握用户入口、企业流程、独家数据，再用 Agent 提升效率，那它的护城河可能会变得更深。

这就是区别。

AI 当然会改变软件，也会改变工作流。未来一定会出现很有价值的 Agent 公司。

但问题是，今天市场上很多 Agent 公司，可能只是站在一轮技术浪潮的前排，还没有证明自己能把技术浪潮沉淀成长期商业模式。

我不是不看好 Agent。

我是看不清大多数 Agent 公司 3 年、5 年、10 年后的竞争格局。

如果你要投资，或者要长期判断这类公司，最关键的问题不是“它现在的 Demo 炫不炫”，也不是“它是不是 AI 原生”，而是那几个老问题：

它靠什么长期留住客户？

它靠什么维持定价权？

它靠什么不被模型公司和平台公司顺手做掉？

它的利润，是来自短期技术差，还是来自长期结构优势？

如果这些问题回答不出来，那就算它今天看起来很像未来，也未必是一门好生意。

AI Agent 是未来的一部分。

但不是每一家 AI Agent 公司，都能分享未来的利润。
